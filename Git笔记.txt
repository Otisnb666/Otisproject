git：分布式版本控制系统
	
· 版本控制
· 团队协作

本地有仓库，在离线的时候也可以进行版本控制
需要进行团队协作，因此有Github远程仓库


在对应文件加下面右键打开git自带终端 

想要什么文件用git管理就在文件里面 （命令git init）会有一个隐藏的文件夹（版本库）

工作区代码先提交到暂存区暂时保存，确认无误后提交到master主分支产生一条记录。（不一定是主分支，可以在自定义分支）

当A已经提交后，发现A中有小bug，可将修改后的A代码放到暂存区，撤掉master中的A到暂存区中，将完整代码提交到master中，最后可将本地库的代码托管到远端库中


托管平台：
	·局域网（自己搭建，团队使用）
		gitlab
	·公网   （公共使用）
		gitlab	
		github
		gitee（码云）




将指定文件添加到暂存区域
git add wenjian.txt
git add .(添加所有文件)

查看暂存区文件
git status
若文件在暂存区，文件名应是绿色

删除暂存区对应文件
git rm --cached wenjian.txt
删除暂存区文件，文件名是红色
